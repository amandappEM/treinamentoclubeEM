## 🚀 Publicar no GitHub Pages — passo a passo

### Caminho mais fácil: via interface web do GitHub (sem terminal)

**1. Criar a conta (se ainda não tem):**
- Acesse [github.com/signup](https://github.com/signup)
- Use um e-mail que você acessa
- Escolha um nome de usuário (esse vai aparecer na URL pública)

**2. Criar o repositório:**
- Acesse [github.com/new](https://github.com/new)
- **Repository name:** `treinamento-clube` (ou outro nome)
- **Visibility:** **Public** ⚠️ (necessário pro GitHub Pages gratuito)
- ⛔ NÃO marque "Add a README file"
- Clique em **Create repository**

**3. Fazer upload dos arquivos:**
- Na página do repositório novo (vazia), clique no link **"uploading an existing file"** (texto em azul no meio da página)
- Arraste pra dentro da janela os 2 arquivos:
  - `index.html`
  - `README.md`
- Role pra baixo e clique em **Commit changes**

**4. Ativar o GitHub Pages:**
- Ainda no seu repositório, clique em **Settings** (no menu de cima)
- No menu lateral esquerdo, clique em **Pages**
- Em **Source**, deixe selecionado **Deploy from a branch**
- Em **Branch**, escolha `main` e mantenha a pasta `/ (root)`
- Clique em **Save**

**5. Aguarde 1–2 minutos:**
- Recarregue a página de Settings → Pages
- Vai aparecer no topo: **"Your site is live at https://seu-usuario.github.io/treinamento-clube/"**
- Esse é o link que você compartilha com o time! 🎉

## ✏️ Atualizar o treinamento depois

Sempre que quiser atualizar o conteúdo (texto, quiz, simulador):

1. No GitHub, abra o repositório
2. Clique no arquivo `index.html`
3. Clique no ícone do lápis ✏️ (canto superior direito) pra editar online
4. Faça as alterações
5. Role pra baixo e clique em **Commit changes**
6. Pronto — a atualização vai pro ar em ~30 segundos automaticamente

Ou, se preferir, apague o `index.html` no repositório e faça upload de uma versão nova.

## 📧 Como você recebe os aproveitamentos

Ao final do treinamento, o CS clica em **"Enviar registro de conclusão"** — abre um e-mail pré-preenchido no cliente padrão da pessoa, com destinatário `amanda.pimentel@estantemagica.com.br`.

O e-mail traz:
- Identificação (nome, e-mail, time, líder)
- Nota da avaliação final (X/12) + Status (Aprovado/Aprovado com Excelência/Necessita Revisão)
- Tempo total de execução
- Performance dos checkpoints intermediários
- Performance dos 5 cenários do simulador (🟢🟡🔴)
- Detalhe das respostas da avaliação final

Caso o cliente de e-mail não abra automaticamente, há um botão **"Copiar registro"** que copia tudo pro clipboard pra ela mandar manualmente.
