# Treinamento Clube Estante Mágica — Time CS

Treinamento interativo do Clube Estante Mágica para o time de Customer Success (Ciclo 2026). Versão V7 com simulador de conversa estilo WhatsApp, calculadora de pontos e avaliação consolidada.

🔗 **URL pública:** [https://amandappem.github.io/treinamentoclubeEM/](https://amandappem.github.io/treinamentoclubeEM/)

---

## 🎯 O que tem dentro do treinamento

**13 módulos** estruturados em 3 partes + avaliação final:

### Parte I — Entenda o Clube
- M1 · Boas-vindas
- M2 · O que é o Clube
- M3 · Quem participa e a faixa
- M4 · As 16 ações que pontuam (5 pilares)
- M5 · Os 4 kits do Clube (Kit Evento + 3 opcionais)
- M6 · Catálogo e calculadora interativa de pontos
- M7 · Operação na prática (envio, prazos, fluxo de exceção)

### Parte II — Você é o agente do MaaS
- M8 · O Clube em cada persona (Diretor/Coord/Prof/Família)
- M9 · Você é o porta-voz do MaaS
- M10 · Como apresentar (scripts prontos)

### Parte III — Dominando o que vier
- M11 · Objeções, dúvidas e o "Kit Mágico continua"
- M12 · 🎯 Simulador de Conversa WhatsApp (5 cenários)
- M13 · Avaliação Final (12 questões)

---

## ✨ Recursos interativos

- **Calculadora de pontos** (M6) — testa combinações de itens por faixa, em tempo real
- **Simulador de conversa estilo WhatsApp** (M12) — 5 cenários com 17 turnos e 51 escolhas possíveis, com reações da escola e encerramento de conversa em respostas inadequadas
- **Checkpoints com quiz** ao final de cada módulo (30+ perguntas)
- **Avaliação final consolidada** com envio de resultado por e-mail
- **Reveal cards** com diagnósticos e cenários práticos
- **Cards "Sabia que?"** com micro-aprendizados

## 🚀 Como publicar (primeira vez)

### 1. Criar repositório no GitHub
1. [github.com/new](https://github.com/new)
2. **Repository name:** `treinamentoclubeEM` (ou outro)
3. **Visibility:** **Public** (necessário pro GitHub Pages gratuito)
4. ⛔ NÃO marque "Add a README file"
5. **Create repository**

### 2. Subir os arquivos
Na página do repositório novo, **uploading an existing file** → arraste `index.html` e `README.md` → **Commit changes**.

### 3. Ativar o GitHub Pages
1. **Settings** → **Pages** (menu lateral)
2. **Source:** *Deploy from a branch*
3. **Branch:** `main` / `(root)` → **Save**
4. Aguarde 1–2 min, recarregue. A URL pública aparece no topo.

## ✏️ Como atualizar uma nova versão

Quando rolar uma versão nova do `index.html`:

1. Acesse o repositório no GitHub
2. Clique em `index.html`
3. Clique no ícone do lápis ✏️ (canto superior direito)
4. **Apaga TODO o conteúdo** (Cmd+A → Delete)
5. **Cola o conteúdo da versão nova**
6. Role pra baixo → **Commit changes** com mensagem do tipo `V7 — feedbacks do time aplicados`
7. ~30s a 1min depois, a URL pública atualiza automaticamente

## 📧 Como você recebe os aproveitamentos

Ao final, o CS clica em **"Enviar registro de conclusão"** → abre o cliente de e-mail pré-preenchido com destinatário **amanda.pimentel@estantemagica.com.br**.

O e-mail traz:
- Identificação (nome, e-mail, time, líder)
- Nota da avaliação final (X/12) + Status (Aprovado / Aprovado com Excelência / Necessita Revisão)
- Tempo total de execução
- Performance dos checkpoints intermediários
- Performance dos 5 cenários do simulador (🟢🟡🔴 ou 🚫 Interrompido)
- Detalhe das respostas da avaliação final

Se o cliente de e-mail não abrir automaticamente, o CS clica em **"Copiar registro"** e cola no Gmail/Outlook web.
